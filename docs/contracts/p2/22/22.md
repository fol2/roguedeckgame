# Contract 22 - Phase 17 Hardening

## Scope

This contract covers gate reliability, submit hardening, and localhost integration robustness for the browser-first TypeScript, Vite, and Phaser pet roguelite deckbuilder.

The implementation does not add gameplay or content. It keeps game rules inside `src/game-core` and presentation/runtime submission concerns inside `src/game-phaser`.

## Requirements

- Default `npm test` must be bounded and must exit cleanly.
- Explicit test lanes must exist for CLI, core, Phaser, and scripts.
- CLI smoke tests must not silently hang.
- Localhost smoke tests must cleanly close server connections.
- Vite preview integration must prove built localhost routes and assets work.
- Browser DOM smoke must be hard by default, including Content Workbench render proof and browser error checks.
- Preview and browser child processes must be stopped reliably.
- Production-facing `RunSandboxController` run-level mutations must require `expectedRevision` and `requestId`.
- Production scene, preview, and test call sites must submit explicit revision and request ids.
- Missing request ids, duplicate request ids, missing revisions, and stale revisions must reject before mutation.
- Current checkout verification and localhost deployment evidence must be captured in this folder.
