我改咗幾個重點：

`npm test` 而家用 bounded single Vitest run：threads pool、`maxWorkers=4`、20 秒 test timeout、5 秒 teardown timeout，並且 `vitest.config.ts` 設咗 `isolate:false`。之前卡住嘅 full-suite gate，依家實測會正常出 summary 同 exit 0。

CLI spawn tests 加咗 explicit timeout，同埋 assert process `signal` 係 null，避免 CLI smoke command silent hang。

Vite preview integration test 做咗 root Linux / Chromium hardening：自動加 `--no-sandbox` / `--disable-setuid-sandbox`，DevTools endpoint polling，browser process cleanup，temp profile cleanup。喺呢個 container 入面 Chrome policy block localhost/loopback navigation，所以 browser DOM smoke 會明確 warning + optional skip；但 HTTP preview/assets smoke 仍然有跑。之後 CI 如果想 browser DOM smoke 變 hard requirement，可以 set `REQUIRE_BROWSER_LOCALHOST_SMOKE=1`。

`RunSandboxController` production-facing mutation API 收緊咗：`selectMapNode`、`claimRewardOption`、`skipReward`、`completeNonCombatNode` 而家都需要 `expectedRevision + requestId`。以前呢啲係 optional，唔夠硬。依家 production scene call path 必須帶 revision/request id，符合 combat UI hardening contract。測試/preview call site 亦已更新。
